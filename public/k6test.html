การทดสอบ Performance ด้วย k6

เราใช้ k6 เพื่อสร้าง **"ทีม user จำลอง"** เข้าไปใช้งานระบบพร้อมกัน เพื่อดูว่าเว็บไซต์ของเราแข็งแกร่งแค่ไหน และจะรับมือไหวไหมเมื่อเจอภาระงานหนักๆ

k6 เป็นเครื่องมือทดสอบประสิทธิภาพ (Performance Testing) แบบ Open-source ที่ทันสมัยและได้รับความนิยมสูงในกลุ่มนักพัฒนาทั่วโลกครับ จุดเด่นคือการเขียนสคริปต์ทดสอบด้วยภาษา JavaScript ซึ่งทำให้นักพัฒนาสามารถสร้างเคสทดสอบที่ซับซ้อนได้อย่างรวดเร็วและง่ายดาย นอกจากนี้ k6 ยังขึ้นชื่อเรื่องความเร็วและประสิทธิภาพของตัวมันเอง ทำให้สามารถจำลองผู้ใช้งานจำนวนมากได้โดยใช้ทรัพยากรเครื่องไม่สูงนัก

รูปแบบการทดสอบ (Scenario)

เราจำลองสถานการณ์ให้เหมือนมี user ใหม่เข้ามาใช้งานระบบอย่างต่อเนื่อง โดยมีเงื่อนไขคือ:

  • เป้าหมาย: มี register เข้ามา 15 คน ต่อวินาที
  • ระยะเวลา: ทดสอบต่อเนื่องเป็นเวลา 60 วินาที
  • User ที่เตรียมไว้: เตรียม user เสมือนไว้สูงสุด 35 คน เพื่อรองรับภาระงาน

เกณฑ์ตัดสิน 'ผ่าน/ตก' (Thresholds)

เราตั้งเกณฑ์เพื่อตัดสินว่าระบบทำงานได้ดีแค่ไหน โดยต้อง:

  • ความเร็ว: 95% ของการ Register ต้องเสร็จใน 3,000ms
  • Error: ยอมรับ Error โดยรวมได้ไม่เกิน 80% (เผื่อกรณีสมัคร user ซ้ำ)
